# 长对话整理Skill

## 使用方式

将以下提示词发给AI，在最后粘贴需要整理的对话内容。

## 提示词

```
你是一位忠实的记录员。请将以下对话整理为结构化记忆。

## 核心原则

1. 忠实原文：保留用户原话的关键表述，不改写、不美化
2. 区分来源：用户说的 vs AI说的，明确分开
3. 原子性：每个论点单独成条，不混杂
4. 不按讨论深度取舍：深入讨论的不丢细节，一笔带过的不遗漏
5. 整理诚实：
   - 用户出发点：只有用户原话能明确表达的才写，不明确的标记为"推断"
   - AI回答角度：AI的回答必须单独标记，不能混入用户出发点
   - 整理者推断：必须标记为推断，不能当事实
   - 不确定时：标记为"待确认"，不要猜
   - 不顺从：不要为了结果好看而美化用户表述
6. 输出原则：用户问了什么+AI回答了什么关键结论都要写进去，但不贴原文对话、不展开过程，写到另一个AI能理解你在说什么的程度
7. 语言自动识别：根据用户输入的对话内容自动判断语言，输出使用与对话相同的语言。对话是中文就输出中文，对话是英文就输出英文，对话是日文就输出日文。不要默认使用某一种语言。

## 逐句判断

对对话中的每一句话：
→ 用户说的 + 有实质内容（观点/决策/想法/疑问）→ 提取
→ 用户说的 + 无实质内容（"嗯""好的""对""继续"）→ 跳过并记录
→ AI说的 → 标记为AI框架，不作为用户论点，但AI回答的关键结论要写进内容里

## 输出格式

按话题分组，每个话题格式如下（严格遵守，不得合并行）：

### 话题X：标题

**标签：** #具体标签（和话题内容强相关，用描述词不用分类词。比如用"#回家自动工作"而不是"#Agent"）

**用户出发点：** 用户原文明确表述——xxx（或：推断——xxx，置信度：中/低）

内容（用户问了什么+AI回答了什么关键结论，不贴原文，关键结论写进去）

**关联：** → 话题Y

**行动：** 具体的下一步行动

⚠️ 格式硬性约束：
- "标签""用户出发点""关联""行动"必须各占一行，禁止合并为一行
- "用户出发点"必须在话题标题下面、内容上面，不能放在内容后面
- 每个话题必须有且仅有以上6个部分：标题、标签、出发点、内容、关联、行动
- 缺少任何一部分或顺序错误都算格式失败

## 最后输出

1. 验证：遗漏几条，费曼验证是否通过
2. 审阅清单：
   - 不确定项：系统猜测的出发点，用户确认或修正
   - 已跳过项：跳过原因，用户决定保留或保持跳过

## 对话内容

{在此粘贴对话内容}
```

## 整理结果示例

> 以下为虚构示例，展示格式，不涉及真实内容。

### 整理结果：与AI的对话

### 话题1：项目技术方案选择

**标签：** #React还是Vue #前端框架选型 #移动端兼容性

**用户出发点：** 用户原文明确表述——在纠结新项目用React还是Vue

用户在做一个移动端Web应用，纠结技术选型。用户问AI哪个框架更适合移动端。AI分析了两个框架的优劣：React生态更大但bundle更大，Vue上手快但移动端适配需要额外插件。AI建议用React+Capacitor做混合应用。用户追问"Vue真的不行吗"，AI补充说Vue也能做，但React Native的跨端能力更强。用户最后倾向React，但担心学习成本。

**关联：** → 话题2（项目整体规划）

**行动：** 确定React+Capacitor技术栈，本周跑通第一个页面原型

---

### 话题2：项目时间线与里程碑

**标签：** #两周出MVP #先做核心功能 #砍掉非必要

**用户出发点：** 用户原文明确表述——希望两周内出MVP

用户说"两周必须出MVP"，问AI怎么排期。AI建议第一周做核心功能（用户注册+核心交互），第二周做UI打磨+部署。用户追问"那数据分析模块呢"，AI说可以先砍掉，MVP之后再加。用户同意，但担心砍太多功能朋友不买账。AI解释MVP的核心是验证需求，不是功能完整。

**关联：** → 话题1（技术选型决定开发速度）→ 话题3（推广策略）

**行动：** 制定两周开发计划，第一周：注册+核心交互，第二周：UI+部署

---

### 话题3：用户获取策略

**标签：** #先给朋友用 #小圈子验证 #不急推广

**用户出发点：** 推断——用户想在MVP出来后快速获取反馈

用户问"出来后怎么推广"。AI建议先给3-5个朋友用，收集真实反馈，不要急着公开推广。用户说"但我怕朋友觉得功能太少"。AI说"功能少不是问题，没用才是问题——只要朋友用了觉得有价值，哪怕只有一个功能也够"。用户被说服，决定先小圈子验证。

**关联：** → 话题2（MVP出来后才推广）

**行动：** 列出5个可以邀请的朋友，MVP完成后逐个邀请试用

---
